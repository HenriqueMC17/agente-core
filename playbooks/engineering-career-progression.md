## Filosofia Core

> "Níveis de carreira são **indicadores atrasados** (lagging indicators). Você não é promovido para **tentar** atuar no próximo nível; você é promovido porque já está operando nele de forma consistente."

```
Progressão ≠ linear em esforço
Progressão = exponencial em IMPACTO

Júnior  → resolve 1 tarefa
Pleno   → resolve 1 feature
Sênior  → resolve o sistema de features
Staff   → resolve o sistema que previne falhas no sistema
```

## A Matriz de Níveis

### Júnior (L1-L2): A Fundação Técnica
```
FOCO:         Transformar requisitos claros em código funcional e limpo
ESCOPO:       Tarefa individual
AUTONOMIA:    Baixa (precisa de orientação frequente)
IMPACTO:      Arquivos individuais

CRITÉRIOS:
  □ Entrega tasks definidas no prazo
  □ Escreve código legível com testes básicos
  □ Pede ajuda antes de ficar bloqueado por > 2h
  □ Comunica progresso proativamente

SINAL PARA PROMOÇÃO:
  → Opera consistentemente em nível Pleno por 3+ meses
  → Resolve tasks sem supervisão constante
```

### Pleno (L3): Independência e Craft
```
FOCO:         Features completas com qualidade e autonomia
ESCOPO:       Feature / Módulo
AUTONOMIA:    Média (define como implementar, não o quê)
IMPACTO:      Módulos e integrações

CRITÉRIOS:
  □ Entrega features completas com testes e documentação
  □ Resolve ambiguidade técnica com pesquisa independente
  □ Identifica e comunica riscos proativamente
  □ Mentora Júniors em code review

SINAL PARA PROMOÇÃO:
  → Toma decisões técnicas corretas sem supervisão
  → Impacto além da própria feature
```

### Sênior (L4-L5): Liderança Técnica
```
FOCO:         Qualidade técnica do time + valor de negócio
ESCOPO:       Projeto / Squad
AUTONOMIA:    Alta (define o quê e o como)
IMPACTO:      Sistemas e squads

CRITÉRIOS:
  □ Resolve problemas de alta ambiguidade com julgamento técnico
  □ Influencia decisões de arquitetura com evidências
  □ Eleva a barra técnica do time (code reviews rigorosos)
  □ Traduz requisitos de negócio em decisões técnicas
  □ Articula trade-offs para stakeholders não técnicos

SINAL PARA PROMOÇÃO:
  → Já opera como Staff Engineer de fato
  → Impacto transcende o squad
```

### Staff Engineer (L6): Multiplicador Estratégico
```
FOCO:         Visão sistêmica + habilitação de outras pessoas
ESCOPO:       Área / Organização
AUTONOMIA:    Máxima (define a direção técnica)
IMPACTO:      Organização inteira

CRITÉRIOS:
  □ Identifica e resolve problemas antes que se tornem crises
  □ Cria alavancas (ferramentas, padrões, processos) que habilitam times
  □ Influência técnica além das fronteiras do time
  □ Conecta estratégia técnica à estratégia de negócio
  □ Cria trilhas de desenvolvimento para Júniors e Plenos

MARCOS CONCRETOS:
  → Liderou mudança arquitetural que impactou múltiplos sistemas
  → Criou ferramenta/padrão adotado por outros times
  → Reduziu MTTR ou melhorou DORA metrics em ≥ 30%
```

| Dimensão | Júnior | Pleno | Sênior | Staff |
|---|---|---|---|---|

## O Trabalhador Nativo de IA (2026)

Em 2026, a progressão inclui uma nova dimensão: **maturidade no uso de IA**:

| Nível | Uso de IA |
|---|---|
| **Iniciante** | Usa IA como chatbot, aceita respostas sem revisão |
| **Intermediário** | Usa IA para acelerar, mas revisa todo output |
| **Avançado** | Orquestra workflows agênticos com COVE e FLARE |
| **Expert** | Projeta sistemas de IA, governa agentes, define políticas |

> "O papel do engenheiro está sendo redefinido: de 'programador de rotina' para 'arquiteto de decisões', onde o foco recai sobre o julgamento estratégico e a governança de sistemas autônomos."

## Guia Prático para Aceleração

### O Que Fazer Agora (nível N → N+1)

```
1. MAPEIE O GAP:
   → Liste 3 comportamentos do nível N+1 que você NÃO faz hoje
   → Essas são suas iniciativas de crescimento

2. CRIE VISIBILIDADE:
   → Trabalho invisível não conta para promoção
   → Documente decisões técnicas em ADRs
   → Apresente em tech talks internos
   → Mentore alguém publicamente

3. OPERE NO NÍVEL ACIMA PRIMEIRO:
   → A promoção é a confirmação, não o começo
   → Assuma responsabilidades do nível desejado antes de pedí-la

4. MEÇA SEU IMPACTO:
   → "Reduzi o tempo de build em 40% (de 8min para 4min)"
   → "Meus code reviews bloquearam 3 bugs críticos em produção"
   → Números > palavras sempre

5. CONSTRUA ATIVOS DURÁVEIS:
   → Código → Arquitetura → Produto → Ativo → Liberdade
   → Todo projeto deve gerar algo reutilizável
```

## DORA Metrics: A Linguagem de Negócio da Engenharia

Conecte seu trabalho às métricas que C-Suite entende:

| Métrica DORA | O Que Mede | Elite (2026) |
|---|---|---|
| **Deployment Frequency** | Velocidade de entrega | Múltiplos por dia |
| **Lead Time for Changes** | Eficiência do pipeline | < 1 hora |
| **Change Failure Rate** | Qualidade do código | < 5% |
| **MTTR** | Resiliência e resposta | < 1 hora |

> Engenheiros Staff são avaliados por como seus sistemas movem estas métricas, não por linhas de código escritas.
